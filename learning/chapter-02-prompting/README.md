# Chapter 02 Prompting

这一章只解决一个问题：为什么 prompt 要模板化。

## 这一章为什么学

如果每次都手写完整提示词，后面一改角色、一改任务、一改输入内容，代码就会越来越乱。模板化的价值，就是把固定结构和可变内容拆开。

## 这一章只讲什么

本章只讲 `PromptTemplate` 和变量替换，不展开聊天消息、不展开结构化输出、不展开 RAG 和 Agent。

## 运行方式

1. 先按 `learning/README.md` 进入本章目录。
2. 运行本章最小示例。
3. 对比不同输入下，模板输出如何变化。

## 本章目标

- 看懂什么是 prompt 模板。
- 看懂变量占位是怎么工作的。
- 理解同一个模板换不同输入时，输出为什么会变化。

## 建议修改点

- 先改模板里的角色描述，再看输出差异。
- 再改变量值，观察内容是否只变化在可变部分。
- 最后试着把一个变量删掉，理解模板依赖哪些输入。
