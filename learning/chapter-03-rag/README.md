# Chapter 03 RAG

这一章只解决一个问题：如何把本地知识放进问答流程里。

## 这一章为什么学

单独问模型时，模型只能靠自身参数里的知识回答。RAG 的价值，是先把本地资料检索出来，再让模型基于资料回答，这样更贴近真实项目里的知识问答场景。

## 这一章只讲什么

本章只讲最小 RAG 闭环：

- 准备本地知识文本
- 切分文档
- 向量化
- 检索相关片段
- 把片段交给模型回答

不展开混合检索、多知识库、rerank 或 Web 项目接入。

## 运行方式

1. 先按 `learning/README.md` 进入本章目录。
2. 运行本章最小示例。
3. 对比“没有检索上下文”和“有检索上下文”时的回答变化。

## 本章目标

- 看懂 RAG 的完整最小流程。
- 理解“先检索，再生成”这件事。
- 能把本地资料接进一个可运行的问答脚本。

## 建议修改点

- 先改 `data/knowledge.txt` 里的内容，再看回答变化。
- 再改问题，观察检索到的片段是否变化。
- 最后调整切分大小，理解分块对检索结果的影响。
