# Chapter 03 RAG

这一章只解决一个问题：**如何把本地知识放进问答流程里。**

## 先记住一句话

RAG = **先检索，再生成**。

模型本身不一定知道你的私有资料，但你可以先把资料找出来，再让模型基于资料回答。

## 这一章为什么学

如果只问模型，答案只能靠模型自己“记得什么”。
如果先把本地知识检索出来，再交给模型，回答会更贴近你的真实资料。

这就是 RAG 的意义。

## 这一章只讲什么

本章只讲最小 RAG 闭环：

1. 准备本地知识文本
2. 把长文本切成小片段
3. 给每个片段做向量化
4. 根据问题找出最相关的片段
5. 把片段拼进提示词，让模型回答

不展开混合检索、多知识库、rerank 或 Web 项目接入。

## 运行方式

1. 先看 `data/knowledge.txt`，知道本章知识来源是什么。
2. 再运行 `demo.py`，观察每一步打印。
3. 对比“检索到的上下文”和“模型最终回答”。

## 本章目标

- 看懂 RAG 的完整最小流程。
- 理解“先检索，再生成”这件事。
- 能把本地资料接进一个可运行的问答脚本。

## 你要重点看什么

- 文档是怎么切分的
- 问题是怎么变成向量的
- 哪几个片段被召回了
- 模型最后是怎么只基于资料回答的

## 建议修改点

- 先改 `data/knowledge.txt` 里的内容，再看回答变化。
- 再改问题，观察检索到的片段是否变化。
- 最后调整切分大小，理解分块对检索结果的影响。
