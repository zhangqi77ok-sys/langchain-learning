# Chapter 04 Memory

这一章只解决一个问题：为什么多轮对话看起来像“记住了前文”。

## 先记住一句话

很多场景下，所谓 memory，本质上不是模型真的长期记住了你说过的话，而是**系统把历史消息再次带回了下一次输入里**。

## 这一章为什么学

你在聊天产品里经常会感觉模型“记住了”前面说过的话，比如：

- 你先说“我最喜欢 Python”
- 下一轮再问“我刚才喜欢什么语言”
- 模型还能答出来

这背后的关键，不一定是什么神秘记忆系统，而往往只是：**上一轮消息被重新传给了模型。**

## 这一章只讲什么

本章只讲最容易理解的 memory 本质：

1. 没有历史消息时会怎样
2. 带上历史消息时会怎样
3. 为什么多轮对话能力，很多时候就是“历史输入再喂一次”

不展开 LangGraph persistence，不展开复杂 memory 组件。

## 运行方式

1. 先按 `learning/README.md` 进入本章目录。
2. 运行本章最小示例。
3. 对比“单轮提问”和“带历史提问”时的输出差异。

## 本章目标

- 理解 memory 最基础的实现思路。
- 看懂历史消息是怎么重新进入模型输入的。
- 能自己写出一个最小多轮对话示例。

## 建议修改点

- 先改第一轮里让模型记住的内容，再看第二轮回答是否变化。
- 再删掉历史消息，观察模型为什么答不出来。
- 最后自己增加第三轮消息，看看上下文是怎么累积的。
