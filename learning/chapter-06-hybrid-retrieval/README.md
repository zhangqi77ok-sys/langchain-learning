# Chapter 06 Hybrid Retrieval

这一章只解决一个问题：**为什么很多检索系统要同时用 BM25 和向量检索。**

## 先记住一句话

混合检索的核心不是“多做一次检索”，而是：

- BM25 负责看关键词
- 向量检索负责看语义
- 最后把两路结果融合起来

你也可以直接把它记成一句更直白的话：

**混合检索本质上就是把“关键词相关性”和“语义相关性”两套分数融合，得到更稳的召回结果。**

## 这一章为什么学

如果只用 BM25，关键词不一致时容易漏召回。
如果只用向量检索，语义虽然能对上，但有时对精确关键词不够敏感。

所以真实系统里，经常把两者放在一起用。

## 这一章只讲什么

本章只讲最小混合检索流程：

1. 准备一组本地文档
2. 用 BM25 检索一次
3. 用向量检索一次
4. 把两路结果做简单融合
5. 对比最终排序

不展开 rerank，不展开多知识库，不展开 Web 项目接入。

## 运行方式

1. 先阅读本章的本地文档内容。
2. 运行本章示例脚本。
3. 对比 BM25、向量检索、融合结果三者的差异。

## 本章目标

- 理解 BM25 在看什么。
- 理解向量检索在看什么。
- 看懂为什么要做融合。
- 能跑通一个最小混合检索 demo。

## 这一章学完后你应该能自己说出来

混合检索不是一个神秘的新东西。
它的本质就是：

1. 先做关键词检索
2. 再做语义检索
3. 然后按权重把两边分数合起来
4. 最后按综合分数重新排序

也就是：

**关键词分 + 语义分 -> 按权重合成总分 -> 重新排序**

## 建议修改点

- 先改查询词，观察三种结果如何变化。
- 再改文档内容，观察关键词命中和语义命中的区别。
- 最后调融合权重，理解最终排序为什么会变。
