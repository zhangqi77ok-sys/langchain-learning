# Chapter 07 LangGraph

这一章只解决一个问题：**LangGraph 到底是在编排什么。**

## 先记住一句话

LangGraph 的核心不是“模型突然更强了”，而是：

- 把一个任务拆成多个节点
- 让状态在节点之间流转
- 按固定顺序或规则往下执行

## 这一章为什么学

前面你已经学了：

- 模型调用
- Prompt
- RAG
- Memory
- Agent

但当流程开始变复杂时，就会出现一个问题：

**这些步骤怎么组织起来？**

LangGraph 解决的就是这个问题。

## 这一章只讲什么

本章只讲最小 LangGraph 闭环：

1. 定义一个共享状态
2. 定义两个节点
3. 让第一个节点写状态
4. 让第二个节点读取前一个节点写入的状态
5. 看最终状态如何形成

不展开条件分支，不展开 checkpoint，不展开复杂 Agent 图。

## 运行方式

1. 先按 `learning/README.md` 进入本章目录。
2. 运行本章最小示例。
3. 顺着打印结果看状态如何一站一站往后传。

## 本章目标

- 理解 LangGraph 最基础的工作方式。
- 看懂什么是节点。
- 看懂什么是状态流转。
- 能跑通一个最小两节点图。

## 建议修改点

- 先改初始输入，再看后续状态如何变化。
- 再新增一个状态字段，观察图如何继续传递。
- 最后自己加第三个节点，理解为什么 LangGraph 适合编排流程。
