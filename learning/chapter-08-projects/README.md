# Chapter 08 Projects

这一章只解决一个问题：**前面学的这些东西，怎么落到项目里。**

## 先记住一句话

前面的章节学的是“零件”，这一章学的是“怎么把零件拼成项目”。

你前面已经学过：

- 最小模型调用
- Prompt 模板
- RAG
- Memory
- Agent
- Hybrid Retrieval
- LangGraph

真正做项目时，不是把它们分开看，而是要开始想：

- 这个项目的入口是什么
- 检索放在哪一层
- 模型回答放在哪一层
- Web 界面和后端怎么配合

## 这一章为什么学

只会跑单个 demo，还不等于会做项目。

项目视角最重要的是：你要开始把“知识点”变成“模块职责”。

比如：

- RAG 是检索问答模块
- Memory 是对话上下文模块
- Agent 是工具决策模块
- LangGraph 是流程编排模块

## 这一章只讲什么

本章不新做一个大项目。

本章只做两件事：

1. 帮你建立项目视角
2. 把当前仓库里已有项目和前面章节对应起来

## 当前项目入口

当前仓库里你已经有一个项目级目录：

- [`projects/multi-kb-rag-web`](</E:/langchain-learning/projects/multi-kb-rag-web>)

这个目录就是后面继续强化的主项目。

## 你可以怎么理解这个项目

可以先把它看成 4 层：

1. **Web 层**
   - 页面输入问题
   - 页面展示答案

2. **应用层**
   - 接收问题
   - 调用检索和模型
   - 组织最终返回

3. **检索层**
   - 加载知识文本
   - 切分
   - 向量化
   - 检索相关内容

4. **模型层**
   - 根据检索上下文生成答案
