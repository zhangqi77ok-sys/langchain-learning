# Chapter 05 Agents

这一章只解决一个问题：**Agent 和普通模型调用到底差在哪。**

## 先记住一句话

普通模型调用通常是：

- 你问
- 模型直接答

而 Agent 更像：

- 你问
- 模型先判断要不要用工具
- 如果要用，就先调工具
- 最后再组织答案

## 这一章为什么学

很多真实问题，模型光靠“自己想”不够。

比如：

- 查天气
- 做计算
- 查数据库
- 查知识库

这时更合理的做法不是让模型瞎猜，而是让模型先决定是否调用外部工具。

## 这一章只讲什么

本章只讲最小 Agent 闭环：

1. 定义两个本地工具
2. 把工具交给 Agent
3. 提一个需要工具参与的问题
4. 看 Agent 最终如何回答

不展开多 Agent，不展开 LangGraph Agent 编排，不展开复杂工具系统。

## 运行方式

1. 先按 `learning/README.md` 进入本章目录。
2. 运行本章最小示例。
3. 观察最终回答和中间消息轨迹。

## 本章目标

- 理解 Agent 和普通模型调用的区别。
- 看懂“模型决定调工具”这件事。
- 能跑通一个最小工具型 Agent。

## 建议修改点

- 先改问题内容，看看 Agent 会不会换工具。
- 再给工具返回不同结果，观察最终回答是否跟着变。
- 最后自己新增一个工具，理解 Agent 为什么能扩展。
