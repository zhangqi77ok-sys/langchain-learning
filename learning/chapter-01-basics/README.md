# Chapter 01 Basics

这一章先解决一个最重要的问题：如何跑通第一次最小模型调用。先把输入、调用、输出这条最短链路理解清楚，后面的 prompt、RAG、Agent 才有共同起点。

## 这一章为什么学

先学最小模型调用，是为了建立最基础的使用感觉：模型怎么接入、请求怎么发、结果怎么拿。只要这一步跑通，后面的内容就不会停留在概念层面。

## 这一章只讲什么

本章只讲“最小模型调用”，不展开 prompt 设计、不展开检索增强、不展开 Agent 编排。目标是先把一次完整调用做对，再考虑扩展。

## 运行方式

1. 先按 `learning/README.md` 的章节顺序进入本章目录。
2. 按本章提供的最小示例运行。
3. 观察模型返回结果，确认输入和输出都能正常闭环。

## 本章目标

- 理解一次最小模型调用的完整流程。
- 能看懂请求里最关键的输入项。
- 能根据本章示例完成一次可运行的调用。

## 建议修改点

- 先改模型名，再看返回差异。
- 再改输入内容，观察输出变化。
- 最后只保留你真正需要的最小参数。
