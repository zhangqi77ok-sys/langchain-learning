# 07-retriever-basics

这一组案例只讲一件事：**为什么 RAG 不是靠关键词匹配，而是让问题和文档都先变成向量，再做相似度检索。**

## 你要学会什么

- 什么是 embedding
- 什么是向量检索
- 为什么问题和文档都要向量化
- `k` 为什么会影响召回结果

## 这个案例的核心流程

1. 准备几个文档块
2. 创建本地 embedding 模型
3. 把文档放进向量存储
4. 用问题做相似度检索
5. 打印最相近的结果

## 你要特别注意

- 这一步还不是最终问答，只是“先找资料”
- 检索质量会影响后面回答质量
- `k` 不是越大越好，也不是越小越好

## 为什么要学这个

因为 RAG 的第一关键步不是生成答案，而是：

**你到底能不能先把最相关的资料找回来。**

如果资料都没召回，再强的模型也很难答好。

## 建议你自己改的地方

1. 改查询问题
2. 改文档内容
3. 改 `k`
4. 观察不同问题召回结果有什么变化

## 这一组案例的目标

让你真正理解：

**RAG 的“检索”阶段，本质是在找“语义上最接近”的文档块。**
