# 09-tool-calling

这一组案例只讲一件事：**为什么模型有时不会直接回答，而是先要求调用工具。**

## 你要学会什么

- 什么是 Tool Calling
- 什么是 `tool_calls`
- 为什么模型第一次响应可能不是最终答案
- 工具结果是怎么再回到模型里的

## 这个案例的核心流程

1. 定义一个本地工具
2. 把工具绑定到模型
3. 把问题发给模型
4. 如果模型触发工具调用，就执行工具
5. 把工具结果作为 `ToolMessage` 交回模型
6. 模型再生成最终回答

## 你要特别注意

- Tool Calling 不是 Agent
- Tool Calling 更像“单次工具辅助回答”
- 模型本身不会真的执行 Python 函数，是框架帮你执行的

## 为什么要学这个

因为后面你做：
- 查询天气
- 查数据库
- 调搜索接口
- 调企业内部服务

都离不开这个闭环。

## 运行方式


## 建议你自己改的地方

1. 改提问城市
2. 改工具返回内容
3. 改成一个工具字典里没有的城市

## 这一组案例的目标

让你真正理解：

**Tool Calling 的核心不是“模型自己会查数据”，而是“模型先决定要不要调用工具，再由系统真正执行工具”。**
