# 04-output-parser

这一组案例只讲一件事：**为什么模型输出有时不能直接拿来给程序用，而需要先经过 parser。**

## 你要学会什么

- 模型调用返回的到底是什么
- 为什么返回值不一定是普通字符串
- `StrOutputParser` 在做什么
- `prompt | model | parser` 这条链路是什么意思

## 这个案例的核心流程

1. 读取环境变量
2. 创建提示词模板
3. 创建聊天模型对象
4. 创建输出解析器
5. 把三者串成一条链
6. 调用并打印最终字符串结果

## 你要特别注意

- 没有 parser 时，模型返回的通常是“消息对象”
- 有 parser 后，输出会被收束成更适合程序处理的结果
- 这一组先只讲“字符串解析”，还不讲结构化对象解析

## 为什么要学这个

因为后面你做：
- 结构化输出
- RAG 结果处理
- Tool 结果再加工
- Agent 最终结果整理

都离不开“输出怎么转换成程序更好处理的形式”。

## 运行方式

在仓库根目录 `E:\langchain-learning` 下执行：

```powershell
.\.venv\Scripts\python fundamentals\04-output-parser\demo.py
```

## 建议你自己改的地方

1. 把 parser 去掉，自己打印返回对象看看
2. 改提问内容
3. 比较“有 parser”和“没 parser”的区别

## 这一组案例的目标

让你真正理解：

**模型输出不只是“显示给人看”，很多时候还要先整理成程序能继续处理的形式。**
