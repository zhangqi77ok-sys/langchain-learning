# 08-rag-basics

这一组案例只讲一件事：**检索和生成是怎么真正串起来，形成最小 RAG 链路的。**

## 你要学会什么

- 为什么 RAG 不是“只有检索”
- 为什么 RAG 也不是“只有模型生成”
- 最小 RAG 到底分成哪几步
- 检索结果为什么要先整理成上下文，再交给模型

## 这个案例的核心流程

1. 准备文档
2. 创建本地 embedding
3. 把文档放进向量存储
4. 用问题做相似度检索
5. 把召回结果拼成上下文
6. 把上下文和问题一起交给模型回答

## 你要特别注意

- 检索阶段的输出不是最终答案
- 检索阶段的作用是“先找资料”
- 生成阶段的作用是“基于资料组织答案”
- 没有检索，RAG 不成立
- 没有生成，也不是完整 RAG

## 为什么要学这个

因为很多人会把 RAG 理解成：
- “向量检索”
或者
- “把文档塞给模型”

这都不完整。

更准确地说：

**RAG = 检索 + 上下文组织 + 基于上下文生成答案**

## 建议你自己改的地方

1. 改文档内容
2. 改查询问题
3. 改 `k`
4. 观察检索上下文变化后，最终答案如何变化

## 这一组案例的目标

让你真正理解：

**RAG 的关键不是“让模型更会说”，而是“先把相关资料找回来，再让模型基于资料说”。**
