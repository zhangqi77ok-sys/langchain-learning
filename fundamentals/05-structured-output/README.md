# 05-structured-output

这一组案例只讲一件事：**为什么有些场景下，普通文本还不够，程序必须拿到固定字段。**

## 你要学会什么

- 什么是结构化输出
- 为什么结构化输出比普通字符串更适合程序处理
- `with_structured_output(...)` 在做什么
- Pydantic 模型为什么会出现在这里

## 这个案例的核心流程

1. 读取环境变量
2. 定义一个固定字段的数据结构
3. 创建聊天模型对象
4. 让模型按这个结构返回结果
5. 读取结构化字段并打印

## 你要特别注意

- 普通文本适合“给人看”
- 结构化输出适合“给程序继续处理”
- 如果后面你要做 API、表单、数据库写入、工作流判断，结构化输出会更重要

## 为什么要学这个

因为后面你做：
- Tool 结果整理
- Agent 最终结果规范化
- Web API 返回固定 JSON
- 复杂项目里的流程判断

都会遇到一个共同问题：

**程序不是想看一段散文，而是想拿到固定字段。**

## 运行方式

```powershell
.\.venv\Scripts\python fundamentals\05-structured-output\demo.py
```

## 建议你自己改的地方

1. 改字段名
2. 改字段描述
3. 改提问内容
4. 观察模型返回对象和普通字符串的区别

## 这一组案例的目标

让你真正理解：

**结构化输出的重点不是“更高级”，而是“更适合程序接着往下处理”。**
