# 02-prompt-template

这一组案例只讲一件事：**为什么要使用 PromptTemplate，而不是把提示词一直写死在字符串里。**

## 你要学会什么

- 什么是提示词模板
- 什么是模板变量
- 为什么模板比写死字符串更适合反复复用
- `prompt | model` 这种链式写法在干什么

## 这个案例的核心流程

1. 读取环境变量
2. 创建一个提示词模板
3. 创建聊天模型对象
4. 把模板和模型串起来
5. 传入变量
6. 打印模型结果

## 你要特别注意

- 这里依然是单轮调用
- 这里还没有消息列表
- 这里也还没有 RAG、Tool、Agent
- 这一组的重点只有“模板化输入”

## 为什么要学这个

如果你一直把提示词写死，后面一旦要换：
- 说话风格
- 目标用户
- 解释对象
- 业务变量

代码会越来越乱。

PromptTemplate 的价值，就是把“固定结构”和“可变部分”拆开。

## 运行方式

在仓库根目录 `E:\langchain-learning` 下执行：

```powershell
.\.venv\Scripts\python fundamentals\02-prompt-template\demo.py
```

## 建议你自己改的地方

1. 改 `style`
2. 改 `target`
3. 改 `topic`
4. 故意少传一个变量，观察报错

## 这一组案例的目标

不是让你记住 API 名字，  
而是让你真正理解：

**提示词模板 = 固定骨架 + 动态变量。**
