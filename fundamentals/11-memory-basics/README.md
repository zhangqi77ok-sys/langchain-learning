# 11-memory-basics

这一组案例只讲一件事：**多轮对话为什么能“记住”前文，以及这种记忆本质上是怎么实现的。**

## 你要学会什么

- 什么是对话历史
- 什么是会话 ID
- 为什么“记忆”通常不是模型自己永久记住了
- 为什么说 Memory 很多时候本质是“历史消息重放”

## 这个案例的核心流程

1. 创建一个提示词模板
2. 在模板中预留历史消息位置
3. 创建聊天模型对象
4. 创建内存中的历史消息存储
5. 用同一个 `session_id` 连续发两轮问题
6. 观察第二轮是否能基于第一轮内容回答

## 你要特别注意

- 这里的“记忆”不是数据库持久化
- 这里的“记忆”也不是模型自己学会了什么
- 这里的核心是：框架把历史消息再次带回模型输入

## 为什么要学这个

因为后面你做：
- 对话型助手
- Web 聊天页面
- 用户历史会话
- 带上下文连续性的知识库问答

都离不开“历史消息怎么进模型”这件事。

## 运行方式

在仓库根目录 `E:\langchain-learning` 下执行：

```powershell
.\.venv\Scripts\python fundamentals\11-memory-basics\demo.py
```

## 建议你自己改的地方

1. 改第一轮让模型记住的内容
2. 改第二轮追问方式
3. 改 `session_id` 看是否还记得

## 这一组案例的目标

让你真正理解：

**所谓 Memory，在很多基础场景里，不是模型多了神奇能力，而是框架把历史消息重新塞回去了。**
