## 当前状态

- 已记录方向
- 已开始补详细学习计划

## 为什么现在学 LangGraph

前面的 LangChain 基础和项目练习，已经让你学会了：

- 模型调用
- Prompt
- RAG
- Tool Calling
- Agent
- Memory

但这些能力在真实项目里如果继续变复杂，就会遇到新的问题：

- 多步流程怎么拆节点
- 分支逻辑怎么管理
- 中间状态怎么保存
- 工具调用后下一步怎么决定
- 失败后怎么恢复

这就是 LangGraph 要解决的问题。

## LangChain 和 LangGraph 的区别

可以先用一个不严格但实用的理解：

- `LangChain`
  更偏“能力组件”和“最小调用方式”

- `LangGraph`
  更偏“工作流编排”和“状态驱动流程”

也就是说：

- LangChain 更像组件层
- LangGraph 更像流程层

## 建议学习顺序

### 第 1 阶段：最小 StateGraph

目标：
- 先理解什么是图
- 先理解节点怎么连接
- 跑通最小二节点流程

你要学会：
- `StateGraph`
- `START`
- `END`
- 最小节点函数

### 第 2 阶段：节点与边

目标：
- 明确每个节点做什么
- 明确边为什么存在

你要学会：
- 普通边
- 节点职责拆分
- 输入输出状态传递

### 第 3 阶段：条件分支

目标：
- 根据中间结果走不同路径

你要学会：
- 条件判断函数
- 条件边
- 分支型工作流

### 第 4 阶段：多节点工作流

目标：
- 从最小图扩展到稍微像真实任务的流程

你要学会：
- 多步串联
- 状态在多节点间传递
- 节点间职责划分

### 第 5 阶段：工具节点

目标：
- 把 Tool Calling 放进图结构里

你要学会：
- 节点里调用工具
- 工具结果回写状态
- 工具调用后的下一步路由

### 第 6 阶段：对话状态

目标：
- 把多轮对话上下文纳入图状态

你要学会：
- 消息历史放进 state
- 对话类工作流如何设计

### 第 7 阶段：checkpoint / persistence

目标：
- 让流程状态可恢复、可延续

你要学会：
- 为什么要 checkpoint
- persistence 解决什么问题
- 为什么这比简单内存 history 更工程化

### 第 8 阶段：LangGraph 与 Agent 的关系

目标：
- 弄清 Agent 只是 LangGraph 能承载的一种模式

你要学会：
- Agent 为什么可以放进图
- 图工作流为什么比单 Agent 更可控
- 什么时候用 Agent，什么时候直接用图
