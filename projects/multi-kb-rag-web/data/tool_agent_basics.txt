Tool Calling 的核心思想是：
让模型在回答问题之前，先判断是否需要借助外部工具。

这些工具可以是：
- 天气查询
- 搜索接口
- 数据库查询
- 代码执行器
- 企业内部服务接口

模型擅长理解问题和组织语言，但并不天然拥有实时外部数据。
因此，当任务依赖外部信息时，工具调用就很重要。

Agent 可以看成建立在 Tool Calling 之上的更高一层抽象。

普通 Tool Calling 一般是：
1. 模型决定调用某个工具
2. 系统执行工具
3. 把工具结果再交回模型
4. 模型输出最终答案

Agent 的特点在于：
- 不一定只调用一次工具
- 可以根据中间结果继续决策
- 可以决定是否调用别的工具
- 在满足条件后再停止

因此，Tool Calling 更适合单次工具辅助回答，Agent 更适合多步骤任务。
